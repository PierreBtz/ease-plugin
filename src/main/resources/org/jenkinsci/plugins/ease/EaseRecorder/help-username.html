Enter the User ID for a valid user in your EASE organization. The user must have valid permissions for adding applications to EASE. Based on this User ID, the EASE web service identifies your organization in the EASE database. For more information on user permissions, see Managing Users (https://help.apperian.com/display/pub/Managing+Users) in the Apperian Documentation Center.

You can leave User ID and Password blank if you have stored valid credentials with the Jenkins Credentials plugin. Note that the plugin matches the domain set for the stored credentials with the domain in the API endpoint.
